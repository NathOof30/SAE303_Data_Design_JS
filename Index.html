Interprétation des données

Comment lire ces visualisations

  • Temps par solveur : Compare la rapidité moyenne. Un temps élevé (≥10000s) signale un timeout.
  • Évolution vs taille : Courbe montante = performance dégrade avec la complexité.
  • Statuts : Satisfaisant = solution trouvée, Insatisfaisant = impossible, Inconnu = timeout/échec.
  • Filtrage : Sélectionnez un benchmark pour comparer les solveurs sur un problème précis.
